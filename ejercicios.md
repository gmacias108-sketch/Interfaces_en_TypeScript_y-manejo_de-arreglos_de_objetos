1. ¿Qué es una interface en TypeScript?

Una interface en TypeScript es una estructura o plantilla que define las propiedades y los tipos de datos que debe tener un objeto. Su función es establecer reglas para que todos los objetos creados a partir de ella tengan la misma organización.

2. ¿Para qué sirve un arreglo de objetos?

Un arreglo de objetos sirve para almacenar varios elementos que tienen la misma estructura. Permite organizar información relacionada, como listas de estudiantes, productos o clientes, y facilita realizar operaciones como buscar, filtrar, agregar o actualizar datos.

3. ¿Qué ventaja tiene usar interfaces?

La principal ventaja de utilizar interfaces es que ayudan a mantener el código organizado y evitan errores al momento de crear objetos. Además, hacen que todos los objetos tengan la misma estructura, lo que facilita el desarrollo y el mantenimiento del programa.

4. ¿Cuál es la diferencia entre find() y filter()?

La diferencia es que find() busca el primer elemento que cumple una condición y devuelve un solo objeto. Si no encuentra ninguno, devuelve undefined.

En cambio, filter() busca todos los elementos que cumplen la condición y devuelve un nuevo arreglo con todos los resultados encontrados. Si no encuentra ninguno, devuelve un arreglo vacío.

5. ¿Por qué TypeScript ayuda a evitar errores en el código?

TypeScript ayuda a evitar errores porque permite definir el tipo de dato que debe tener cada variable, función u objeto. De esta manera, detecta muchos errores durante el desarrollo antes de ejecutar el programa, haciendo que el código sea más seguro, organizado y fácil de mantener.
